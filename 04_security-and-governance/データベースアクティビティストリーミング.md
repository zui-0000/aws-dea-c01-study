# データベースアクティビティストリーミング（Database Activity Streams）

## 概要

Amazon Aurora / RDS の**データベース操作をリアルタイムでキャプチャ**し、
Kinesis Data Streams に流すセキュリティ・監査機能。

「誰が・いつ・どんなSQLを実行したか」をリアルタイムで記録・監視できる。

```
Aurora / RDS
    ↓ SQL実行・ログイン・エラーなどの操作が発生
Database Activity Streams（キャプチャ）
    ↓
Kinesis Data Streams（ストリームに流す）
    ↓
Lambda / パートナーツール（分析・監視・アラート）
```

---

## DynamoDB Streams との比較

DynamoDB Streamsの「RDS/Aurora版」に近いイメージだが、**目的が根本的に違う**。

```
DynamoDB Streams          →「データの変更」をキャプチャ（何が変わったか）
Database Activity Streams →「DB操作」をキャプチャ（誰が何をしたか）
```

| | Database Activity Streams | DynamoDB Streams |
|--|--|--|
| 主目的 | 監査・セキュリティ | データ連携・イベント処理 |
| キャプチャ内容 | 誰がいつ何をしたか（操作） | データが何に変わったか（変更） |

---

## 向いている業界

コンプライアンス要件（監査ログの保存義務）がある業界で特に使われる。

| 業界 | 理由 |
|------|------|
| **金融** | 「誰がいつ口座情報を参照したか」の証跡が法的に必要 |
| **医療** | 患者データへのアクセス記録（HIPAA準拠） |
| **公共・政府** | 個人情報DBの操作履歴の保管義務 |

> 試験で「コンプライアンス・監査・規制業界」というキーワードが出たら Database Activity Streams を思い出す。

---

## DynamoDB Streams との違い（詳細）

似た名前だが用途が全然違う。

| 観点 | Database Activity Streams | DynamoDB Streams |
|------|--------------------------|-----------------|
| 対象DB | Aurora / RDS | DynamoDB |
| キャプチャ内容 | DB操作（SQL・ログイン・エラー） | データの変更（追加・更新・削除） |
| 主な用途 | セキュリティ監査・コンプライアンス | データ連携・イベント処理 |
| 出力先 | Kinesis Data Streams | Lambda など |

---

## キャプチャできる内容

- SQLクエリ（SELECT・INSERT・UPDATE・DELETE）
- ログイン・ログアウト
- エラー・失敗したクエリ
- DDL操作（テーブル作成・削除など）

---

## 動作モード

| モード | 説明 | 特徴 |
|--------|------|------|
| **非同期モード** | DB操作とキャプチャが非同期 | DBパフォーマンスへの影響が少ない |
| **同期モード** | DB操作と同時にキャプチャ | データの欠損なし・パフォーマンスへの影響あり |

> **重要（モードの選択可否）:**
> - 同期/非同期の**両モードを選択できるのは Aurora のみ**。
> - **RDS（Oracle / SQL Server）の DAS は常に非同期（asynchronous）**で、同期モードは選択できない。

---

## ユースケース

### ① コンプライアンス・監査

```
「誰がいつ個人情報テーブルにアクセスしたか」を記録
    ↓ Database Activity Streams
Kinesis → S3 に保存（監査ログとして保管）
```

### ② セキュリティ脅威の検知

```
不審なSQLが実行される（大量のSELECT・深夜のアクセスなど）
    ↓ Database Activity Streams
Kinesis → Lambda → アラート通知
```

### ③ パートナーツールとの連携

AWS公認のセキュリティパートナーツールと連携して高度な監視ができる。
例: IBM Security Guardium、McAfee MVISION

---

## 対応サービス

Database Activity Streams（DAS）に対応しているのは **Aurora MySQL / Aurora PostgreSQL / RDS for Oracle / RDS for SQL Server の4つのみ**。RDS for MySQL・RDS for PostgreSQL・RDS for MariaDB・RDS for Db2 は**非対応**。

| サービス | 対応 |
|---------|------|
| Amazon Aurora MySQL | ✅ |
| Amazon Aurora PostgreSQL | ✅ |
| Amazon RDS for Oracle | ✅ |
| Amazon RDS for SQL Server | ✅ |
| Amazon RDS for MySQL | ✗（**非対応**） |
| Amazon RDS for PostgreSQL | ✗（**非対応**） |
| Amazon RDS for MariaDB | ✗（**非対応**） |
| Amazon RDS for Db2 | ✗（**非対応**） |
| Amazon DynamoDB | ✗（DynamoDB Streams を使う） |

> **注意:** 「RDS for MySQL」は非対応。MySQL系で DAS を使えるのは **Aurora MySQL** のみ。

---

## 試験のポイント

- **DB操作の監査・セキュリティ監視** → Database Activity Streams
- **出力先は Kinesis Data Streams**（Firehoseではない）
- **DynamoDB Streams との違い** → DASはSQL操作の監査、DynamoDB Streamsはデータ変更の連携
- **コンプライアンス要件**（誰がいつ何を操作したか証明したい）でよく登場
- **対応エンジン** → **Aurora MySQL / Aurora PostgreSQL / RDS for Oracle / RDS for SQL Server のみ**。**RDS for MySQL は非対応**（MySQL系は Aurora MySQL のみ対応）
- **同期/非同期** → **Aurora は同期/非同期を選択可、RDS（Oracle / SQL Server）は常に非同期**。同期モードはデータ欠損なし、非同期モードはパフォーマンス影響が少ない
