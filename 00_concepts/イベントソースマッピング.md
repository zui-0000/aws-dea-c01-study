# イベントソースマッピング（Event Source Mapping）

## 概要

**「イベントが発生したら自動でLambdaを起動する」仕組み**。
AWSのイベントソース（Kinesisなど）とLambdaを繋ぐ設定のこと。

Kinesisの機能でもLambdaの機能でもなく、**2つのサービスの間をつなぐ接着剤（橋）**。

```
┌─────────────────────────────────────┐
│        イベントソースマッピング          │
│                                     │
│  Kinesis  ←→ポーリング←→  Lambda   │
│                                     │
└─────────────────────────────────────┘
```

設定はLambdaの設定画面から行う。

```
Lambdaの設定画面で
「このLambdaのトリガー元はKinesis（このストリーム）にする」
    ↓
AWS側が自動でポーリングの仕組みを作ってくれる
```

---

## バッチ処理との比較

経験のあるバッチ処理と比べると違いがよくわかる。

```
【バッチ処理（スケジュール起動）】
EventBridgeスケジュール
    ↓「毎日0時に起動して！」（時間ベース）
Lambda / Glue / Step Functions
    ↓
データを処理

【イベントソースマッピング（イベント起動）】
イベントソースマッピング
    ↓「データが来たら即起動して！」（データベース）
Lambda
    ↓
データを処理
```

| 観点 | スケジュール起動（EventBridge） | イベントソースマッピング |
|------|-------------------------------|----------------------|
| 起動タイミング | 時間で指定 | データ到着時 |
| リアルタイム性 | 低い（次の実行時間まで待つ） | 高い（即時） |
| 用途 | 定期バッチ・日次集計 | ストリーミング処理 |

---

## 仕組み

```
イベントソース（Kinesis など）にデータ到着
    ↑
Lambda が裏でイベントソースをポーリング（AWS が自動管理）
    ↓ 新しいレコードを検知！
Lambda が自動起動
    ↓
処理
```

ポーリング（監視）はAWS側が全部やってくれるので、自分で実装する必要はない。

### ポーリングのイメージ

一定間隔でKinesisに「データ来てますか？」と問い合わせ続ける。

```
Lambda（AWSが管理）

───────────────────────────────────→ 時間

↓確認   ↓確認   ↓確認   ↓確認   ↓確認
Kinesis Kinesis Kinesis Kinesis Kinesis
「ない」 「ない」 「ある！」「ある！」「ない」
                  ↓       ↓
                Lambda   Lambda
                起動！    起動！
```

もし自分でポーリングを実装するとしたらこうなるが、イベントソースマッピングを設定するだけでAWSが全部やってくれる。

```python
# 自前でやる場合（実際には不要）
while True:
    records = kinesis.get_records()
    if records:
        lambda_function(records)
    time.sleep(1)  # 1秒ごとに確認
```

**「設定するだけでリアルタイム処理が動く」** のがイベントソースマッピングの強み。

---

## 対応しているイベントソース

| イベントソース | 説明 |
|-------------|------|
| **Amazon Kinesis Data Streams** | ストリームデータをリアルタイムで処理 |
| **Amazon DynamoDB Streams** | DynamoDBの変更をリアルタイムで処理 |
| **Amazon SQS** | キューにメッセージが来たら処理 |
| **Amazon MSK / Kafka** | Kafkaトピックのメッセージを処理 |

---

## バッチサイズ

一度にLambdaに渡すレコードの数を設定できる。

```
バッチサイズ = 100 の場合：
Kinesisに100件溜まる（または一定時間経過）
    ↓
Lambdaに100件まとめて渡す
    ↓
一度の処理で100件をまとめて捌く
```

---

## 試験のポイント

- **「データが来たら即起動」** → イベントソースマッピング
- **「決まった時間に起動」** → EventBridgeスケジュール
- **Kinesis → Lambda の連携** = イベントソースマッピングが必要
- ポーリングはAWSが自動管理（自分で実装不要）
