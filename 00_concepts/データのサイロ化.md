# データのサイロ化（Data Silos）

## 概要

組織内のデータが部門やシステムごとにバラバラに孤立して保存されており、
横断的にアクセスや分析ができない状態のこと。

## 具体例

```
営業部門      → 独自のCRMにデータ
マーケ部門    → 別のツールにデータ
財務部門      → また別のDBにデータ
         ↓
お互いのデータが見えない・使えない！
```

## 問題点

- データを統合して分析できない
- 同じデータが複数箇所に重複して存在する（一貫性が失われる）
- 意思決定が遅くなる・精度が落ちる
- データガバナンスが複雑になる

## AWSにおける解決策

サイロ化を解消するアーキテクチャが **データレイク**。

| サービス | 役割 |
|---------|------|
| **Amazon S3** | 全データを一か所に集める（データレイクの中核） |
| **AWS Lake Formation** | データレイクの構築・管理・アクセス制御 |
| **AWS Glue** | バラバラなデータの変換・統合（ETL） |
| **AWS DMS** | 既存DBからデータを移行 |
| **AWS AppFlow** | SaaSアプリからデータを取り込む |

## 試験のポイント

- サイロ化の解消 → **データレイク構築** がキーワード
- データレイクの中核は **Amazon S3**
- アクセス制御・カタログ管理は **AWS Lake Formation**
- ETL処理は **AWS Glue**
