# <img src="./images/aws-data-exchange.png" width="36"/>&nbsp;&nbsp; AWS Data Exchange

## 概要

**サードパーティのデータをAWS上でサブスクリプション契約して利用できるマーケットプレイス**。
外部企業が提供するデータセットを購入・契約し、直接S3やRedshiftに取り込める。

```
データプロバイダー（外部企業）
├── 気象データ会社
├── 金融データ会社
├── 人口統計データ会社
├── 医療データ会社
└── マーケティングデータ会社
        ↓ AWS Data Exchange（マーケットプレイス）
データコンシューマー（自社）
        ↓ S3に自動配信
Amazon S3 → Athena / Redshift で分析
```

---

## 仕組み

```
① AWS Data Exchange のカタログからデータを検索
② サブスクリプションを申し込む（有料・無料）
③ データがS3に自動的に配信される
④ 新しいデータが更新されると自動で同期される
⑤ AthenやRedshiftで分析
```

自分でAPIを叩いたりデータを取得するコードを書く必要がない。

---

## データの種類

| カテゴリ | 例 |
|---------|-----|
| **金融・経済** | 株価・為替・経済指標 |
| **気象・環境** | 天気予報・気温・降水量 |
| **人口統計** | 国勢調査データ・地域統計 |
| **医療・ライフサイエンス** | 臨床データ・ゲノムデータ |
| **マーケティング** | 消費者行動・購買データ |
| **地理空間** | 地図・住所・POIデータ |

---

## フィルタリング・変換機能

**Data Exchange自体に変換・フィルタ機能はない。** S3に届いてから別サービスで処理する。

```
AppFlow       → 転送しながら変換・フィルタができる
Data Exchange → 変換・フィルタなし。S3に届いてから別サービスで処理
```

```
AWS Data Exchange
    ↓ そのままS3に届く（加工なし）
Amazon S3
    ↓
├── AWS Glue（ETL・変換）
├── Amazon Athena（SQLでフィルタ・集計）
└── Amazon Redshift（分析）
```

---

## Terraformサポート

AppFlowと違い、**Terraformサポートは限定的**。

| 用途 | Terraform |
|------|-----------|
| データを**販売する側**（Provider）の設定 | ✅ 可能 |
| データを**購入する側**のサブスクリプション申込 | △ 主にコンソールUI |

サブスクリプション申込はAWS Marketplaceの購入フロー（規約同意・支払い設定）が絡むため、Terraformだけでは完結しにくい。

```
① コンソールでサブスクリプション申し込み（手動）
② S3への配信設定（コンソール or Terraform）
③ 以降のデータ活用（Glue・Athenaなど）はTerraformで管理
```

---

## AppFlow との違い

似ているようで用途が違う。

| 観点 | AWS Data Exchange | Amazon AppFlow |
|------|-----------------|---------------|
| データソース | **外部企業が販売するデータ** | 自社が使うSaaSアプリ |
| 契約方法 | サブスクリプション購入 | API接続設定 |
| 用途 | 外部データの購入・利用 | 自社SaaSデータの取り込み |

```
「外部企業の気象データを買って分析したい」 → Data Exchange
「自社のSalesforceデータをS3に取り込みたい」→ AppFlow
```

---

## データプロバイダーとしても使える

自社のデータを販売する側（プロバイダー）としても利用できる。

```
自社データ → AWS Data Exchange に出品 → 他社が購入・利用
```

---

## ユースケース

| ユースケース | 説明 |
|------------|------|
| **需要予測** | 気象データ × 自社売上データで需要を予測 |
| **リスク分析** | 金融データ × 自社ポートフォリオでリスク評価 |
| **マーケティング分析** | 人口統計データ × 自社顧客データでターゲティング |
| **新薬開発** | 外部の臨床データを活用した研究 |

---

## 試験のポイント

- **外部のサードパーティデータを利用したい** → AWS Data Exchange
- **データはS3に自動配信** → そのままAthena・Redshiftで分析可能
- **サブスクリプションモデル** → 購入したデータは自動更新される
- **AppFlowとの違い** → Data Exchangeは外部データの購入、AppFlowは自社SaaS連携
- **データプロバイダーとしても使える** → 自社データの販売も可能
