# AWS SCT（Schema Conversion Tool）
<!-- SCTはローカルにインストールするツールであり、AWSマネージドサービスではないため専用アイコンは存在しない -->

## 概要

異種DB移行のときに**スキーマ・SQLコードを自動変換**するツール。
DMSと組み合わせて使う。SCT単体ではデータ移行はできない。

```
SCT  → スキーマ・SQLの変換（構造）
DMS  → データの移行（中身）
```

---

## DMSとの役割分担

```
【異種移行の流れ】

① SCT でスキーマ変換
  Oracle のテーブル定義・ストアドプロシージャ・ビュー
      ↓ 自動変換
  Aurora PostgreSQL 用のDDL・SQLに変換

② DMS でデータ移行
  Oracle（稼働中）
      ↓ フルロード + CDC
  Aurora PostgreSQL
```

SCT → DMS の順番で使う。

---

## 変換できるもの

| 変換対象 | 説明 |
|---------|------|
| **テーブル定義（DDL）** | CREATE TABLE などの構造定義 |
| **ビュー** | VIEW の定義 |
| **ストアドプロシージャ** | DB内のプログラム処理 |
| **関数** | ユーザー定義関数 |
| **インデックス** | 検索高速化のインデックス定義 |

---

## 変換率と手動対応

SCTは**自動変換率をレポートとして出力**してくれる。
100%自動変換できないケースもあり、変換できない部分は手動対応が必要。

```
SCT 変換レポート例：
  自動変換可能: 85%
  手動対応が必要: 15%  ← ここは自分で書き直す
```

変換が難しいケース：
- DBベンダー固有の関数（Oracle独自関数など）
- 複雑なストアドプロシージャ
- 非標準のSQL構文

---

## 同種移行では不要

エンジンが同じなら変換の必要がないためSCTは使わない。

```
【SCT が必要（異種移行）】
Oracle           → Aurora PostgreSQL
SQL Server       → Aurora MySQL
Aurora MySQL     → Aurora PostgreSQL  ← AWS内でもエンジンが違えばSCT必要

【SCT が不要（同種移行）】
MySQL            → RDS MySQL
Aurora MySQL     → RDS MySQL
PostgreSQL       → Aurora PostgreSQL
```

---

## インストールと動作

SCTはAWSのマネージドサービスではなく、**ローカルPC にインストールして使うツール**。

```
① SCTをPC/サーバーにインストール
② ソースDBに接続して分析
③ 変換レポートを確認
④ スキーマをターゲットDBに合わせて変換・出力
⑤ DMSでデータ移行
```

---

## 試験のポイント

- **異種移行（エンジンが違う）** → SCT + DMS のセット
- **同種移行（エンジンが同じ）** → DMS のみ（SCT不要）
- **SCTはスキーマ変換のみ**（データ移行はDMSの仕事）
- **自動変換率100%とは限らない** → 手動対応が必要な部分がある
- **SCTはローカルツール**（マネージドサービスではない）
